## üîë Key Concepts Explained

### Federated Architecture
**Where**: ARCHITECTURE.md (Principles) + README.md (Architecture)
**What**: Hub-and-spoke model where country instances sync with master
**Why**: Data sovereignty, government partnerships, multiple revenue streams

### White-Label
**Where**: README.md (Business Model) + ARCHITECTURE.md (Country Instances)
**What**: Customizable platform per country (logo, colors, content)
**Why**: Tourism boards want their own branded platform

### Adaptive Navigation
**Where**: NAVIGATION_SPEC.md (Navigation Structure)
**What**: Menu changes based on platform mode (Pan-African vs Country)
**Why**: Different user needs on bridge55.com vs bridge55.ci

### Multi-Tenant
**Where**: ARCHITECTURE.md (Database Architecture)
**What**: Single codebase serving multiple country instances
**Why**: Easier to maintain, deploy updates once
