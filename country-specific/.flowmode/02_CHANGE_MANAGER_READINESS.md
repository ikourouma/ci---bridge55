# Change Manager Readiness

**Mandatory for Blast Radius >= 3**

---

## Purpose

Prevent technically correct changes that fail operationally (confusion, resistance, support burden, broken adoption).

---

## When Required

Change Manager readiness is required if:

- ✅ Blast Radius >= 3
- ✅ A user workflow changes (routing, navigation, booking steps, search flow)
- ✅ Any cross-team handoff changes (content ops, support, partners)
- ✅ Any external trust boundary is crossed (partner/regulator/end-user impacts)
